Erstellen Aufgabe 3e — DOM: Liste erzeugen & einfügen

Aufgabe: Eine <ul> mit einem <li>, dessen Attribut land="de" ist, erzeugen und in ein bestehendes <div id="inhalt"> einfügen.

Baue das DOM Schritt für Schritt auf.
Live-DOM in div#inhalt
leer
Resultierendes HTML

✅ Musterlösung (mehrere gültige Wege) 
const ul = document.createElement("ul");
const li = document.createElement("li");
li.setAttribute("land", "de");
ul.appendChild(li);
document.getElementById("inhalt").appendChild(ul);

Alternativen: li.setAttribute("land","de") ≈ direktes Property gibt es für land nicht (kein Standardattribut) → setAttribute ist hier Pflicht. Einfügen via appendChild oder append. Reihenfolge egal, solange am Ende eingehängt wird.

Falle: land ist kein Standard-HTML-Attribut → li.land = "de" setzt nur eine JS-Property, NICHT das HTML-Attribut. Für die Klausur-Aufgabe setAttribute("land","de") verwenden.

[Tracen] Wäre x2 mit new Calculator('Hallo',c) immer noch 1401?

Nein → dann wäre x2 = {"m":1401,"n":"Hallo Welt"} (Objekt). Mit new wird der primitive return ignoriert.

[Tracen] Angenommen die Funktion hätte return {wert:a} statt return a. Was wäre x2 dann?

{"wert":1401}. Ein Plain-Call gibt den Objekt-return zurück. Und x1 wäre dann ebenfalls {"wert":1401} — denn new gibt ein explizit zurückgegebenes Objekt aus (überschreibt das this-Objekt!). Nur primitive returns ignoriert new.
