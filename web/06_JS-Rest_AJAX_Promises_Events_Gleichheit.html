Tracen / MC Gleichheit: == vs. ===

2019-Aufgabe: if (a == 42) … gibt „a ist 42" aus — wegen automatischer Typumwandlung bei ==.

AusdruckErgebniswarum
42 == "42"true== wandelt String→Number
42 === "42"false=== prüft auch den Typ → number ≠ string
0 == falsetruefalse → 0
"" == falsetruebeide → 0
null == undefinedtrueSonderregel von ==
null === undefinedfalseverschiedene Typen
[] == falsetrue[]→""→0, false→0
Regel: == wandelt Typen um (loose), === nicht (strict). In der Praxis fast immer === nutzen. Werte gegen Node verifiziert.

Erstellen AJAX — XMLHttpRequest vs. Fetch

2019-Aufgabe: GET-Request an http://example.org?param=value, Antworttext in answer speichern.

Fetch (modern)

const res = await fetch("http://example.org?param=value");
const answer = await res.text();
// answer enthält den Antworttext

Ohne await mit .then(): fetch(url).then(r=>r.text()).then(a=>answer=a)

XMLHttpRequest (klassisch)

const xhr = new XMLHttpRequest();
xhr.open("GET", "http://example.org?param=value");
xhr.onload = () => { let answer = xhr.responseText; };
xhr.send();
Kontrast: Fetch ist Promise-basiert (async/await, sauberer). XHR ist callback-basiert (älter, umständlicher). Beide: GET-Parameter direkt an die URL anhängen.

Beschreiben Promises & async/await

function ladeDaten() {
  return new Promise((resolve, reject) => {
    setTimeout(() => resolve("fertig"), 100);   // erfolgreich
  });
}
// Verbrauch mit .then / .catch:
ladeDaten().then(d => console.log(d)).catch(e => …);
// oder mit async/await:
async function main() { const d = await ladeDaten(); }

Promise-Zustände: pending → fulfilled (resolve) oder rejected (reject). await pausiert bis das Promise erfüllt ist (nur in async-Funktionen).

Kontrast Arrow Functions & ES6-Classes

Arrow vs. normale Funktion

// normal
function add(a,b){ return a+b; }
// arrow (kurz)
const add = (a,b) => a+b;

Arrow hat kein eigenes this — übernimmt das this der Umgebung. Wichtig bei Callbacks.

ES6-Class vs. Konstruktor-Funktion

class Auto {
  constructor(marke){ this.marke = marke; }
  hupe(){ return "tuut"; }
}
const a = new Auto("VW");

Syntax-Zucker über Prototypen. new Pflicht (sonst Fehler).
