Sicherheit & Datenschutz (D1–D4)

Restthema — Folien „Sicherheit und Privacy". Niedrige Klausur-Prio bei Gen A, aber Grundbegriffe sicher können.

D1 Typische Web-Angriffe

AngriffPrinzipSchutz
XSS
(Cross-Site Scripting)		Angreifer schleust fremdes JavaScript in eine Seite ein (z.B. über ein ungefiltertes Eingabefeld), das im Browser anderer Nutzer läuft.Eingaben escapen/validieren, HttpOnly-Cookies, Content-Security-Policy
CSRF
(Cross-Site Request Forgery)		Eine fremde Seite löst im Namen des eingeloggten Nutzers ungewollt Aktionen aus (Cookie wird automatisch mitgeschickt).CSRF-Token, SameSite-Cookies
SQL-InjectionSchad-SQL wird über Eingabefelder in eine Datenbankabfrage eingeschleust.Prepared Statements / Parameterbindung, Eingaben validieren
// XSS-anfällig: element.innerHTML = userInput; // sicher: element.textContent = userInput; // kein HTML/JS-Interpretieren

D2 HTTPS / TLS

HTTPS = HTTP über TLS (verschlüsselte Transportschicht). Bietet:

VertraulichkeitDaten verschlüsselt → niemand kann mitlesen
IntegritätManipulation unterwegs wird erkannt
AuthentizitätZertifikat beweist, dass der Server echt ist

Ablauf grob: TLS-Handshake (Zertifikat prüfen, Schlüssel aushandeln) → danach verschlüsselte HTTP-Kommunikation.

D4 Sichere Cookies

Securenur über HTTPS gesendet

Set-Cookie: id=abc; HttpOnly; Secure; SameSite=Strict

D3 Datenschutz/DSGVO (Kurz): personenbezogene Daten nur mit Rechtsgrundlage/Einwilligung verarbeiten; Datensparsamkeit; Auskunfts-/Löschrecht. Cookie-Banner für nicht-essenzielle Cookies.

Drill

[MC] Welches Cookie-Attribut schützt gegen XSS-Cookie-Diebstahl?

HttpOnly — JavaScript kann das Cookie dann nicht auslesen.

[Beschreiben] Was macht ein Angreifer bei SQL-Injection?

Er schleust SQL-Code über ein Eingabefeld in eine Datenbankabfrage ein. Schutz: Prepared Statements.

[Nennen] Drei Eigenschaften, die HTTPS/TLS bietet?

Vertraulichkeit (Verschlüsselung), Integrität (Manipulationsschutz), Authentizität (Server-Zertifikat).
