Falle 1 — exakt vs. Wortliste: [class="awe"] (exakt) ≠ [class~="awe"] (enthält das Wort). Bei Mehrfachklassen entscheidend.
Falle 2 — fehlender Doppelpunkt: ol:first-child = Pseudoklasse. ol first-child = Deszendant-Kombinator + Tag-Name → sucht <first-child>-Tag → 0 Treffer. Klassischer Fallenselektor.
Falle 3 — # vs .: #awe sucht eine ID, nicht die Klasse. Kein Element hat die ID awe.
Falle 4 — Teilstring-Attributselektoren: [a^="x"] beginnt · [a$="x"] endet · [a*="x"] enthält (beliebiger Teilstring, KEINE Wortgrenze — ~= dagegen matcht nur ganze, leerzeichengetrennte Wörter). Aus der Dozenten-Zusammenfassung; [class^="…"] kam in Altklausur 2019 dran.
Falle 5 — Geschwister-Kombinatoren: E + F = nur der DIREKTE nächste Nachbar · E ~ F = ALLE nachfolgenden Geschwister.

Von innen nach außen: content → padding → border → margin. Standard box-sizing:content-box: width/height = nur Content. Gesamtbreite = width + 2·padding + 2·border + 2·margin.

Tracen Aufgabe 2e — top/left vom Seitenrand

Klick „Schritt", um blau → grün → rot einzublenden und die berechnete Position zu sehen.

#blue  { position:absolute; left:10px; top:10px; border:5px; w/h:100px }
#green { position:relative; left:10px; top:10px; border:5px; w/h:100px }
#red   { left:10px; top:10px; border:5px; w/h:100px }  /* kein position! */

Falle — border:5px ist wirkungslos! Die Border-Kurzform braucht einen style (z.B. border:5px solid). border:5px allein → border-style:none → keine Border, 0px. Deshalb verschiebt die "5px" nichts.
Falle — #red hat kein position: Default ist static → top/left werden ignoriert. Da #blue absolut (aus dem Fluss) ist, rutscht rot an den Anfang → (0,0).
Falle — absolute vs relative: absolute = relativ zum nächsten positionierten Vorfahren (sonst Viewport), aus dem Fluss. relative = verschoben relativ zur EIGENEN Normalposition, bleibt im Fluss.

Drill

[Tracen] Welche IDs matcht .maria?

Nur p4 (einziges Element mit Klasse maria).

[MC] Matcht ol[class="awe"] das Element p4 (class="awe maria")?

Nein — = verlangt den exakten Wert "awe". Für Teiltreffer bräuchte man [class~="awe"]. Außerdem ist p4 kein ol.

[Tracen] Wäre #green bei position:absolute statt relative, wo läge es?

Dann relativ zum nächsten positionierten Vorfahren = #blue (absolut). left/top:10 → (10,10) INNERHALB von blue → Seitenposition (10+10, 10+10) = (20,20). Gleiche Zahl hier zufällig, aber andere Begründung (Bezug = blue statt Normalfluss).
