CSS-Kaskade & Spezifität (C2)

Restthema — kam in Generation B (Röthig 2016, „Unbekannt II"). Frage: „In welcher Farbe wird der Text angezeigt?"

Regel Reihenfolge der Kaskade

Bei Konflikt entscheidet die Kaskade in dieser Reihenfolge (oben = gewinnt):

RangQuelle
1 (stärkste)User-Stylesheet mit !important
2Author-Stylesheet mit !important
3Author-Stylesheet (normal)
4User-Stylesheet (normal)
5 (schwächste)User-Agent / Browser-Default

Merke: !important dreht User ↔ Author um. Ohne !important gewinnt der Author über den User; MIT !important gewinnt der User über den Author.

Regel Spezifität (innerhalb gleicher Quelle/Wichtigkeit)

Selektor-TeilGewicht
Inline-Style style="…"1 0 0 0
ID #id0 1 0 0
Klasse / Attribut / Pseudo-Klasse .c [a] :hover0 0 1 0
Element / Pseudo-Element div ::before0 0 0 1

Höhere Spezifität gewinnt. Bei Gleichstand: die zuletzt notierte Regel gewinnt (Quellreihenfolge).

Falle: !important schlägt jede Spezifität. Inline-Style schlägt jede normale Regel, aber nicht !important.

Tracen Durchgerechnetes Beispiel (Unbekannt II)

Author (mystyle.css)

body { color:#ffffff; background-color:#000000; }
h1   { color:#0000FF !important; font-size:40px; }
table { color:orange; }

User-Stylesheet

body { color:#FF0000 !important; }
h1   { color:#00FF00; }

Ergebnis

ElementErgebniswarum
body TextrotUser !important schlägt Author-normal
body BGschwarznur Author setzt es
h1 TextblauAuthor !important schlägt User-normal
h1 Größe40pxnur Author
tableorangenur Author
Kernlogik: Beide !important kollidieren nur beim body-Text → dort gewinnt der User (Rang 1). Beim h1 hat nur der Author !important → Author gewinnt.

Drill

[Tracen] Author p {color:blue}, User p {color:green !important} — Farbe?

Grün — User-!important (Rang 1) schlägt Author-normal (Rang 3).

[Tracen] #x {color:red} vs .x {color:blue} am selben Element — Farbe?

Rot — ID (0 1 0 0) hat höhere Spezifität als Klasse (0 0 1 0).

[MC] Was schlägt einen Inline-Style style="color:red"?

Nur eine Regel mit !important. Normale ID/Klasse/Element-Regeln nicht.
