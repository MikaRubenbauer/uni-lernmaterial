SEO — Suchmaschinenoptimierung

Restthema aus Web Engineering_Day6.pdf. Laut Dozent das einzige klausurrelevante Thema aus Day6 (WCAG, Responsive, Core Web Vitals, i18n, Entwurfsmuster sind NICHT dran).

Was ist SEO? Maßnahmen, damit eine Website in Suchmaschinen (Google & Co.) besser gefunden und höher gerankt wird. Drei Hebel: On-Page Technisch Off-Page.

Beschreiben On-Page-SEO (Inhalt & Struktur)

MaßnahmeWarum
Semantisches HTMLSaubere Überschriften-Hierarchie <h1>→<h6>, ein <h1> pro Seite; Struktur-Tags (header/nav/main/article) → Suchmaschine versteht Inhalt.
<title>Wichtigstes On-Page-Signal; erscheint als Klick-Titel im Suchergebnis.
<meta name="description">Snippet-Text unter dem Titel im Suchergebnis (kein direkter Ranking-Faktor, aber Klickrate).
Sprechende URLs/produkte/laptop statt /p?id=42.
alt-Texte für BilderBilder sind sonst für Suchmaschinen „unsichtbar"; auch Barrierefreiheit.
Interne VerlinkungVerteilt „Link-Kraft", hilft Crawlern, alle Seiten zu finden.

Beschreiben Technisches SEO

Datei/TagFunktion
robots.txtSagt Crawlern, welche Bereiche sie (nicht) crawlen dürfen.
sitemap.xmlListe aller URLs für Suchmaschinen → schnellere/vollständige Indexierung.
rel="canonical"Benennt die „Haupt-URL" → vermeidet Duplicate-Content-Abwertung.
JSON-LD / Schema.orgStrukturierte Daten → Rich Snippets (Sterne, Preise, FAQ).
hreflangKennzeichnet Sprach-/Ländervarianten einer Seite.

Beispiele

# robots.txt User-agent: * Disallow: /admin/ Sitemap: https://x.de/sitemap.xml
<link rel="canonical" href="https://x.de/artikel"> <link rel="alternate" hreflang="en" href="https://x.de/en/article">

Beschreiben Off-Page-SEO

Backlinks — Verweise anderer (glaubwürdiger) Websites auf die eigene Seite. Gelten als Empfehlung/Reputationssignal → höheres Ranking. Qualität > Quantität.
Abgrenzung: robots.txt steuert das Crawlen (Zugriff), <meta name="robots" content="noindex"> steuert das Indexieren (Aufnahme in den Index) — nicht verwechseln.
Klausur-Fokus: Day6 ist sonst NICHT relevant — nur SEO. Kernbegriffe sicher benennen können (title/description, robots.txt, sitemap.xml, canonical, Backlinks).

Drill

[Nennen] Zwei On-Page-SEO-Maßnahmen?

z.B. aussagekräftiger <title> + semantische Überschriften-Hierarchie (oder alt-Texte, meta description, sprechende URLs).

[Erklären] Wofür dient rel="canonical"?

Benennt die bevorzugte URL bei mehreren gleichen/ähnlichen Seiten → vermeidet Duplicate-Content-Probleme.

[Kontrast] robots.txt vs. sitemap.xml?

robots.txt = was darf gecrawlt werden (Verbote); sitemap.xml = welche URLs es gibt (Einladung zur Indexierung).
