1 · Max-Heap — Konzept

Ein Max-Heap ist ein vollständiger Binärbaum (von links aufgefüllt), in dem jeder Knoten ≥ seine Kinder ist → die Wurzel ist das Maximum. Gespeichert wird er als Array (0-basiert):

Kind links von i = 2i+1 · Kind rechts von i = 2i+2 · Elter von i = ⌊(i−1)/2⌋

Die Breitensuche/Level-Order eines Baums ist genau diese Array-Darstellung: Ebene für Ebene, links nach rechts.

2 · Heap-Rezepte

R1 — siftDown (versickern)

Element an Position $i$ mit dem größeren seiner Kinder tauschen, solange ein Kind größer ist. Wandert nach unten.

R2 — Build-Max-Heap (Array → Heap)

siftDown auf alle inneren Knoten, von hinten nach vorne: $i = \lfloor n/2\rfloor - 1$ herunter bis $0$. Nach jedem inneren Knoten Zwischenzustand angeben (Array + Baum).

R3 — Heap-Sort

  1. Erst Build-Max-Heap.
  2. Wurzel (Maximum) mit letztem Heap-Element tauschen → größtes steht hinten fest.
  3. Heap-Größe um 1 verkleinern, siftDown auf die (neue) Wurzel.
  4. Schritte 2–3 wiederholen, bis Heap-Größe 1. Ergebnis: aufsteigend sortiert, in-place.
Warum aufsteigend? Der Max-Heap liefert immer das aktuelle Maximum, das ans Ende wandert — so füllt sich das Array von hinten mit den größten Werten.

5 · Huffman & Kompression — Konzept

Huffman erzeugt einen präfixfreien Code minimaler mittlerer Länge: häufige Zeichen → kurze Codes.

  1. Alle Zeichen als Blätter mit ihrer Häufigkeit.
  2. Wiederholt die zwei kleinsten Häufigkeiten zu einem neuen Knoten (Summe) verschmelzen.
  3. Bis ein Wurzelknoten bleibt. Kanten mit 0 (links) / 1 (rechts) beschriften → Code = Pfad Wurzel→Blatt.
Kenngrößen. Entropie $H(Q) = -\sum p_i \log_2 p_i$ (untere Schranke, Bit/Zeichen). Mittlere Codelänge $L = \sum p_i \ell_i$. Redundanz $= L - H$. Bei Speicherung in festen 8 Bit: Redundanz $= 8 - H$ pro Zeichen.
Codes sind nicht eindeutig (0/1-Vertauschung, Gleichstände) — aber die Codelängen je Zeichen und damit die Gesamtlänge sind eindeutig. In der Klausur einen konsistenten Baum zeichnen und die Codetabelle dazu.

10 · Selbsttest

11 · Stolpersteine

  1. Build-Max-Heap von hinten (Index $\lfloor n/2\rfloor{-}1$ herunter bis 0), nicht von vorne.
  2. siftDown tauscht mit dem GRÖSSEREN Kind — sonst bricht die Heap-Eigenschaft.
  3. Heap-Sort: nach jedem Tausch Heap-Größe verkleinern; das fixierte Ende gehört nicht mehr zum Heap.
  4. Array-Indizes 0-basiert ($2i{+}1$, $2i{+}2$) — Prof nutzt 0-Basis. Konsistent bleiben.
  5. Huffman: immer die zwei kleinsten verschmelzen (auch die neuen Summenknoten mitzählen).
  6. Präfixfrei: kein Code ist Anfang eines anderen — automatisch, weil Zeichen nur an Blättern sitzen.
  7. Alle Zwischenzustände angeben (Heap-Aufbau + jeder Heap-Sort-Schritt) — dafür gibt es die Punkte.
